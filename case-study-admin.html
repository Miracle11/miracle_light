Wellness Beauty Admin Dashboard

A scalable internal tool for managing users, businesses, and payments.

View in Figma
Lead UI/UX Designer 2‑week sprint Figma · Tailwind

Overview

Wellness Beauty is a fast-growing platform connecting customers with beauty and wellness service providers (spas, salons, barbershops). As the user base and business registrations expanded, internal admins struggled to manage users, approve new businesses, track bookings, and monitor payments efficiently. There was no unified interface—data was scattered across different tools and manual processes.

Approval made easy

The Challenge

Design a comprehensive, scalable admin dashboard that empowers administrators to:

  • Monitor platform health through key metrics and activity feeds.
  • Manage users (create, edit, suspend, delete) with advanced filtering.
  • Review and approve/reject business registrations quickly.
  • Maintain service categories with proper usage warnings.
  • Gain visibility into bookings and payment flows (including escrow and disputes).

All while keeping the interface clean, intuitive, and ready for future feature expansion.

My Process

1. Research & Discovery

I started by analyzing the existing API documentation to understand data models, endpoints, and admin workflows. I interviewed a few internal admins to learn their pain points: too many clicks to find information, lack of real-time updates, and no easy way to handle bulk actions. This informed the information architecture and feature prioritization.

2. Information Architecture

I structured the dashboard around six core modules:

  • Dashboard: Overview metrics, recent activity, quick actions.
  • User Management: List, view, create, edit, suspend, delete.
  • Business Management: Pending approvals, approve/reject flows, detailed business profiles.
  • Service Categories: CRUD operations with usage indicators.
  • Bookings Management (planned): Filterable list with status tracking.
  • Payments Management (planned): Revenue charts, escrow status, transaction details.

3. Wireframing & Prototyping

Low-fidelity wireframes helped validate layout density and navigation. I focused on:

  • A persistent sidebar for module navigation.
  • Consistent table components with sortable columns and filter bars.
  • Modal-based actions to keep context (approve/reject, create/edit, details).
  • Status badges with color coding for immediate recognition.

Early wireframes exploring layout and navigation.

4. Visual Design

I chose a modern, clean aesthetic using Tailwind CSS. Key decisions:

  • Color Palette: Indigo and purple gradients for branding, with semantic colors for statuses (green=active/success, amber=pending/warning, red=suspended/error).
  • Typography: Inter font for excellent readability at various sizes.
  • Components: Cards with subtle shadows and hover effects, data tables with row highlighting, modals with clear hierarchies.
  • Real-time cues: Badge counts on pending approvals, live timestamps, simulated activity feed.

Final visual design – dashboard with key metrics.

5. Handling “Coming Soon” Features

Since some modules (bookings, payments) were planned but not yet fully implemented, I designed them with placeholder data and clear “Coming Soon” badges. This sets proper expectations while showing the future vision.

Key Design Solutions

Unified Dashboard

Real‑time metrics (total users, pending approvals, revenue) give admins a snapshot at a glance. The activity feed lists recent registrations and bookings for quick follow‑up.

Streamlined User Management

A powerful filter bar (role, status) and global search let admins find anyone instantly. Action buttons (view, edit, suspend, delete) are always visible.

Business Approval Workflow

Cards display business and owner details side by side. Approve/reject modals capture required reasons (for rejection) and trigger email notifications—designed to reduce errors.

Service Categories with Safeguards

When deleting a category, a usage warning appears if services are still linked, preventing accidental data loss.

Payments Dashboard

Charts and escrow statuses give admins financial oversight. The manual release button is disabled but hinted as “coming soon” to align with development roadmap.

Learnings

  • Designing for admins requires balancing data density with usability—too much information can overwhelm, too little can hide critical actions.
  • Involving real users early (even just two admins) uncovered workflow nuances that documentation alone couldn’t provide.
  • Planning for future features with “coming soon” states maintains momentum and aligns stakeholder expectations.
